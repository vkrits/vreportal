This is a query builder web tool that provides assistance in constructing advanced search queries. It follows a general pattern where the end-user wants to search for some entity that he/she knows nothing about, by providing information about other entities for which he/she knows something.

Here are the general steps when using this tool:

1.      This is the first view, where we are only able to select what we are looking for.

2.      Selecting a “target” entity from the drop down list, will enable more options.

3.      The general concept is that we are seeking for target entity (something that we don’t know yet), considering that it has some relation with some other entity (we name that “Related entity”). Relation and “related” entity can be set through the right panel which is divided into two smaller sub-panels for each one respectively.

4.      The system allows specifying actual instances for the related entity.  While searching for “Related Entity” instances, an autocomplete service that recommends search terms is enabled.

5.      Search-terms can be selected from the autocomplete drop-down list. Then by pressing the “” icon, the system searches for relevant instances to display.

6.      The results of this short search appear on a separate dialog on the screen and are split in pages.

7.      Specific instances can be selected by checking the respective check-boxes. The dialog can be closed when the desired instances have been selected by pressing the “Select” button.

8.      The selected instances are listed on the right side of the panel. Removing any of them can be done by clicking on the respective cross icon.

9.      In case that the related entity “under inspection” has geospatial nature, a map can be used for geographically restricting the results while seeking for instances. The map can be opened by pressing on the “” icon.

10.   The region of interesting can be drown by holding the “Ctrl” key and dragging on the map, forming a rectangle. To close the map just click anywhere outside the dialog.

11.   Dropped pins denote instances of the related entity under investigation. In the picture below, the data used are synthetic and thus the geographical coordinates used for the pins are fictional. Since an earlier search was applied by using keywords before, these keywords are also applied as a filter on the map. Restrict results without using any other filters (keyword search terms) but only the map, can be done by closing the map, erasing any keyword search teerms and reopening the map.

12.   Hovering over a pin will reveal information about what it represents by popping a caption above the pin. The caption can be hidden by hovering on another pin or by clicking anywhere on the map but a pin.

13.   Selecting instances can be achieved by clicking on the pins. Clicking on an already clicked pin will de-select the instance.

14.   As soon as the desired instances have been selected, the map can be closed by clicking anywhere outside the dialog. The selected instances will appear on the right side panel similar as previously, along with the previously selected instances.

15.   Adding more than one filters on the “target” entity is possible by clicking on the “” button and selecting “OR” or “AND” to define whether the new filter will be applied uniting or intersecting the results of the first one.

16.   The new filter is displayed below the first one and consists of only the right panel where relation and related entity have to be defined the same way as described before.

17.   Filters can also be applied on the related entities themselves. That is achieved by clicking on the “”.The new filter is displayed below the respective “related” entity and is shifted right a little bit. Once again relation and related entity have to be defined.

18.   Finally as soon as the desired filters have been defined and the finally query has been properly described, we can run it by pressing the “Search” button (this functionality is not yet available)
